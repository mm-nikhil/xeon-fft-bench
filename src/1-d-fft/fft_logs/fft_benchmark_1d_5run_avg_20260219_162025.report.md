## Aggregation Method

- Input source: raw `RESULT|` and `SKIP|` lines from each log.
- Average latency (`ms`): arithmetic mean over successful samples.
- GFLOPS derivation: recomputed from averaged latency for each row.
  `avg_sp_gflops = (5 * N * log2(N) * batch) / (avg_ms * 1e6)`
- For each row key: `profile + workload + case + length + batch + threads`.
- Throughput comparison:
  `Fwd Speedup = avg_fwd_sp_gflops(profile) / avg_fwd_sp_gflops(baseline_sse42_1t)`

  `Bwd % Increase = (Bwd Speedup - 1) * 100`
- GFLOPS columns are single-precision throughput (`SP GFLOPS`).
- Small-size multithread caveat: rows with very small work items (for example, low `N` and low batch) are overhead-dominated and should not be used for thread-scaling conclusions.
